# AIMAX002 — 지게차 관리 (차계부 형식)

서버 없이 브라우저에만 저장되는 정적 PWA입니다. 데이터는 사용하는 스마트폰(브라우저)의 `localStorage`에 저장되며, 서버/DB가 필요 없습니다. 단일 장비(지게차 1대) 기준으로 날짜별 기록을 관리합니다.

## 화면 구성

- **요약**: 상단 날짜 선택 기준 당일 KPI(사용시간/주행거리/주유량/정비건수), 계기 현황(현재·전일 시간계/거리계), 당일 정비·주유 내역
- **사용**: 전일 기준값(직전 기록의 시간계/거리계) 대비, 오늘의 시간계(h)·거리계(km) 누적값을 입력하면 당일 사용량이 자동 계산되어 저장됩니다
- **주유**: 주유량·단가 입력 시 금액 자동 계산 후 저장. 상세 입력이 번거로울 땐 "주유함만 체크" 체크박스로 값 없이 기록만 남길 수 있음
- **정비**: 정비 종류/상세/담당자/비용 기록
- **추이**: 선택한 날짜 기준 최근 7일간 사용시간·주행거리 추이 차트 (Chart.js)
- **⚙️ 설정(헤더 톱니바퀴)**: 장비 이름 등록(헤더 타이틀에 반영), 데이터 백업 내보내기(JSON)/가져오기(복원)

## 데이터 보관 주의사항

모든 데이터는 접속한 기기의 브라우저에만 저장됩니다. 브라우저 데이터(사이트 데이터)를 지우면 기록이 함께 사라지므로, **헤더의 ⚙️ 버튼 → 백업 내보내기**로 주기적으로 JSON 백업을 받아두는 것을 권장합니다. 백업 파일은 같은 메뉴의 "백업 가져오기"로 복원할 수 있습니다.

## Vercel 배포

이 저장소는 순수 정적 파일(`index.html`, `manifest.json`, `sw.js`, `static/`)로만 구성되어 있어 빌드 과정 없이 바로 배포됩니다.

- Repository: `lycian5/AIMAX002`
- Project Name: `aimax-002` (URL: `https://aimax-002.vercel.app`)
- Framework Preset: Other
- Build/Install Command: 비활성화 (`vercel.json`에서 처리)
- Output Directory: `.` (저장소 루트)

배포 후 스마트폰 브라우저로 `https://aimax-002.vercel.app` 접속 → 공유 메뉴에서 "홈 화면에 추가"를 하면 앱처럼 바로 실행할 수 있습니다(PWA, 오프라인 지원).

## 설치형 APK 만들기 (PWABuilder)

브라우저/URL 접속 방식과는 별개로, 실제 설치 파일(APK)이 필요하면 별도 개발환경 설치 없이 아래 방법으로 만들 수 있습니다.

1. `https://aimax-002.vercel.app`가 최신 버전으로 배포되어 있는지 확인
2. https://www.pwabuilder.com 접속 → 주소창에 `https://aimax-002.vercel.app` 입력 후 분석 시작
3. "Android" 패키지 생성 선택 → 옵션 기본값으로 진행하면 서명된 APK(또는 AAB) 파일이 다운로드됨
4. 다운로드된 `.apk` 파일을 스마트폰으로 옮겨 설치 (출처를 알 수 없는 앱 설치 허용 필요)

이 방식으로 만든 앱은 최초 실행 시 한 번은 네트워크로 최신 콘텐츠를 받아오고, 이후에는 서비스워커 캐시 덕분에 오프라인에서도 계속 사용할 수 있습니다. 기존 웹(PWA) 접속 방식은 그대로 유지되며, APK는 별도의 설치형 옵션으로 추가되는 것입니다.

## 로컬 실행

```bash
python -m http.server 8080
```

브라우저에서 `http://127.0.0.1:8080` 접속.
