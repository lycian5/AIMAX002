# BuildNote - 모바일 장비 관리 MVP

중장비 기사님이 휴대폰에서 장비 사용, 작업, 주유, 정비 기록을 빠르게 남기는 로컬 우선 PWA입니다. 빌드 과정이나 서버 없이 정적 파일만으로 실행되며, 인터넷 연결이 불안정한 현장에서도 사용할 수 있습니다.

## 1차 MVP 기능

- 복수 장비 등록 및 전환
- 장비 분류, 규격, 장비 번호 관리
- 누적 아워메타와 주행거리 기록
- 당일 작업시간과 작업 사진 기록
- 주유량, 단가, 금액 및 간편 주유 체크
- 정비 유형, 상세, 비용, 담당자, 다음 점검일 기록
- 장비별 통합 이력과 최근 7일 사용시간
- 오프라인 실행을 위한 PWA 캐시
- JSON 백업 및 복원
- 기존 단일 지게차 기록의 자동 데이터 마이그레이션

## 1.5차 통합관리 모드

- `장비 기록`과 `통합관리` 모드 전환
- 선택 날짜 기준 전체·운행·운휴 장비 현황
- 장비별 사용·작업 기록의 완료, 부분 입력, 미입력 상태 표시
- 다음 점검일 임박·초과 및 기록 누락 알림
- 전체 장비의 작업시간, 주유량, 정비비 요약
- 장비·기록 종류·월별 통합 이력 필터
- 전체 장비 기록 CSV 내보내기
- 장비별 운행 중·운휴 상태 관리

## 1.6차 카카오톡 기록 제출

- 장비별 하루 사용·작업·주유·정비 기록을 한 건으로 취합
- 사용·작업 필수 기록의 입력·미입력 상태 확인
- 제출번호와 수정본(`R2`, `R3`) 자동 생성
- 휴대폰 공유 화면을 통한 카카오톡 오픈채팅 제출
- 공유 기능을 사용할 수 없을 때 제출 내용 복사
- 기록 수정 시 `다시 공유 필요` 상태 자동 표시
- `공유 열림`과 실제 전송 완료를 구분해 안내

## 1.6차 무료 플랜 안내

- 무료 이용 범위: 장비 1대의 기본 기록과 카카오톡 제출
- 설정 화면에서 현재 플랜, 등록 장비 수, 로컬 저장 방식 안내
- 두 번째 장비 추가 시 향후 확장 플랜 범위를 먼저 안내
- MVP 검증 중에는 기존 다중 장비 기능을 강제로 차단하지 않고 계속 테스트 가능
- 통합관리 화면에 이 기기 기록만 취합된다는 범위 표시
- 최초 이용 시 브라우저 데이터 삭제·기기 변경에 대비한 백업 안내

## 스마트폰 홈 화면 설치

- Android Chrome의 설치 이벤트가 준비되면 앱 안에서 설치 창 실행
- iPhone Safari의 공유 메뉴 → `홈 화면에 추가` 절차 안내
- 설치 완료 및 이미 설치된 상태 감지
- 설치 가능한 경우 상단 📲 버튼에 알림 점 표시
- 설치 창이 지원되지 않을 때 기기별 수동 설치 방법과 주소 복사 제공

## 데이터 보관 주의

기록과 첨부 사진은 현재 브라우저의 `localStorage`에 저장됩니다. 브라우저 데이터를 삭제하거나 기기를 변경하면 기록이 사라질 수 있으므로 설정 메뉴에서 정기적으로 JSON 백업을 내려받으세요.

통합관리 모드는 현재 한 휴대폰 안에 저장된 여러 장비를 관리하는 로컬 관리자 화면입니다. 여러 사용자·조직 간 공유, 권한 승인, 클라우드 동기화, 로그인, 거래처 장부, 청구 및 미수금 관리는 후속 단계 범위입니다.

## 로컬 실행

```bash
python -m http.server 8080
```

브라우저에서 `http://127.0.0.1:8080`으로 접속합니다.

## 배포

Vercel 등 정적 호스팅에 저장소 루트 전체를 배포하면 됩니다. 별도 설치·빌드 명령은 필요하지 않습니다.
